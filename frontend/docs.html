Overview

What is this dataset?

The Online Shoppers Intention dataset was collected from an e-commerce platform over one year. Each row represents a single browsing session, and the goal is to predict whether that session will end in a purchase.

Source
Kaggle / UCI ML Repo
by Henry Sue
Task Type
Binary Classification
Predict Revenue (0 or 1)
Imbalance
84.5% vs 15.5%
Handled via balanced weights
⚠️

Class Imbalance Note

Only ~15.5% of sessions result in a purchase. This imbalance was addressed using class_weight="balanced" in tree-based models and optimizing for PR-AUC (Precision-Recall Area Under Curve) instead of accuracy, which is a more informative metric for imbalanced datasets.

About PageValues

PageValues is technically a data leakage candidate — it's computed using transactional data that only exists after a purchase. In real-time prediction, this value may be 0 for all sessions (since no transaction has occurred yet). For demo purposes the API accepts it, and non-zero values will strongly push the prediction toward "Will Purchase".

Feature Engineering

Engineered Features

These features were derived from the raw inputs to capture richer behavioral signals. They are computed automatically inside the model pipeline — you do not need to provide them as inputs.

TotalDuration
= Admin_Duration + Info_Duration + Product_Duration
Total time the visitor spent on the website during the session. Users spending more total time generally show higher purchase engagement.
TotalPages
= Administrative + Informational + ProductRelated
Total number of pages visited. Users who explore many pages are likely in an active decision-making or comparison phase.
ProductDurationPerPage
= ProductRelated_Duration / ProductRelated
Average time per product page. Deep dwell time per product page is a strong signal of genuine purchase consideration.
ProductEngagement
= ProductRelated × ProductRelated_Duration
Combined engagement score measuring breadth (pages visited) × depth (time spent). A high score is the strongest composite purchase signal.
ExitBounceGap
= ExitRates − BounceRates
Measures how much exit behavior exceeds single-page bounce behavior. A positive gap indicates users are leaving after exploring, not just bouncing immediately.
DurationWeightedExit
= TotalDuration × ExitRates
Combines session length with exit tendency. Long sessions with high exit rates may indicate research without intent to buy.
ProductPageRatio
= ProductRelated / TotalPages
Proportion of the session focused on product pages. Higher ratios indicate a product-browsing session rather than an informational one.
Log-transformed Durations
= log1p(duration_feature)
Duration features are highly right-skewed with extreme outliers. Log1p transformation compresses the scale and stabilizes model training.

How Raw Data Becomes a Prediction

The model uses a sklearn Pipeline that automatically handles imputation, encoding, and scaling before feeding data to the classifier.

Feature Engineering

TotalDuration, TotalPages, ratios, engagement scores, and log-transformed duration features are derived from the raw 17 inputs. This step adds 20+ additional features that help the model capture non-linear behavioral patterns.

Missing Value Imputation

Numerical columns use median imputation (robust to outliers). Categorical columns use most-frequent imputation. The dataset had a small percentage of missing values in numeric features only.

SimpleImputer(strategy="median")SimpleImputer(strategy="most_frequent")

Numerical Scaling

RobustScaler is used instead of StandardScaler because duration and page-count features contain significant outliers. RobustScaler centers using the median and scales using the IQR, making it resistant to extreme values.

RobustScalerOutlier-resistantIQR-based scaling

Categorical Encoding

OneHotEncoder with min_frequency=10 is applied to Month, VisitorType, Weekend, and the integer-code columns (OperatingSystems, Browser, Region, TrafficType). Rare categories are grouped as "infrequent."

Why PR-AUC over Accuracy?

With 84.5% of sessions being non-purchases, a naive model predicting "no purchase" every time would achieve 84.5% accuracy — but be completely useless. PR-AUC measures performance specifically on the minority (purchase) class, penalizing models that simply predict the majority class and rewarding those that correctly identify true buyers.

Output Interpretation

Threshold & Decision Boundary

The model outputs a probability score between 0 and 1. A decision threshold converts this into a binary prediction.

Decision Boundary at 0.49

0.49 threshold
0.0 — Will Not Purchase 1.0 — Will Purchase
Probability ≥ 0.49
Will Purchase
Strong enough evidence of purchase intent detected in the browsing session.
Probability < 0.49
Will Not Purchase
Insufficient purchase signals — visitor may need re-engagement or retargeting.
🎯

Why 0.49 instead of the default 0.50?

The threshold was tuned by sweeping values from 0.05 to 0.90 and selecting the threshold that maximizes the F1-score on the purchase class. At 0.49 the model achieves the best balance between Precision (72%) and Recall (64%), slightly favoring capturing more true buyers at the cost of a few extra false alarms.

API Usage

Calling the Prediction API

Send a POST request with a JSON body containing all 17 input fields. The API returns a probability score, label, and threshold.

POST /predict
fetch('https://xstynwx-online-shoppers-api.hf.space/predict', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    "Administrative": 2,
    "Administrative_Duration": 0,
    "Informational": 0,
    "Informational_Duration": 0,
    "ProductRelated": 15,
    "ProductRelated_Duration": 1020.5,
    "BounceRates": 0.3,
    "ExitRates": 0.05,
    "PageValues": 125.3,
    "SpecialDay": 0,
    "Month": "Nov",
    "OperatingSystems": 2,
    "Browser": 4,
    "Region": 1,
    "TrafficType": 2,
    "VisitorType": "Returning_Visitor",
    "Weekend": true
  })
})
// Response:
{
  "will_purchase": true,
  "prediction": 1,
  "prediction_label": "Will Purchase",
  "probability": 0.8134,
  "threshold": 0.49
}
